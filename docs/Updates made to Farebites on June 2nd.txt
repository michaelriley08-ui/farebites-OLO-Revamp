### 📡 GET `/api/Account/profile` (Limitations & Workaround)
* **API Issue:** Returns user profile data but excludes address fields. When the page reloads or a user logs in, fetching profile details from the server clears out the address fields.
* **Workaround:** 
  1. On successful `PUT /api/Account/profile` (update), write the updated address fields to `localStorage`:
     ```javascript
     const localAddressKey = `farebites_profile_address_${email.toLowerCase()}`;
     localStorage.setItem(localAddressKey, JSON.stringify({ address, city, state, zipCode }));
     ```
  2. On page load (`DOMContentLoaded`) and inside the `handleLogin()` callback, fetch the user profile from the server, read the key from `localStorage`, and merge the address details before writing to `mockupState.userProfile`.

### 📡 PUT `/api/Account/profile` (Payload Sanitization)
* **API Issue:** The profile update endpoint expects `UpdateProfileRequest` with `additionalProperties: false`. Sending the `email` field in the request body results in a `400 Bad Request` server validation error.
* **Sanitization:** Sanitized the PUT payload structure to exclude `email`:
  ```javascript
  const payload = { firstName, lastName, phoneNumber, address, city, state, zipCode };
  await window.ApiService.updateProfile(payload);
  ```

### 📡 GET `/api/Account/orders` (Data Normalization)
